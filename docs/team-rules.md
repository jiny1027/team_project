## 팀 규칙

### 1. 브랜치 규칙

- main 브랜치에는 직접 작업하지 않는다.
- 모든 작업은 feature 브랜치에서 진행한다.
- 브랜치 이름은 이슈 타입/#이슈번호-간단한 타이틀 로 통일한다.

---

### 2. 이슈 생성 규칙

- 이슈 탬플릿에 맞춰서 내용을 작성한다.
- [이슈 타입] 간단한 제목

  예시: [DOCS] 한식 디렉토리 구조 형성

---

### 3. 커밋 규칙

- 커밋 메시지는 명확하게 작성한다.
- 형식:
  - feat: 기능 추가
  - fix: 오류 수정
  - docs: 문서 수정

  예시: [docs] 자기소개 수정

---

### 4. 작업 및 Push 규칙

- 작업 완료 후 반드시 본인 브랜치에 push한다.
- main 브랜치에는 직접 push하지 않는다.

---

### 5. Pull Request(PR) 규칙

- 작업 완료 후 PR을 생성한다.
- PR 제목과 내용은 작업 내용을 간단히 작성한다.
- 최소 1명 이상의 리뷰 후 merge 진행

---

### 6. 충돌 해결 규칙

- 충돌 발생 시 main 브랜치를 기준으로 rebase 한다.
- 충돌 파일은 이전 내용이 사라지지 않도록 주의한다.

---

### 7. 코드 리뷰 규칙

- 서로의 PR에 최소 1개 이상의 코멘트를 남긴다.
